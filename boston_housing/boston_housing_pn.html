Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer:

If a predictive model is made based on an entire set of existing data, it is not possible to know if the model can actually predict new data accurately, or if the model can only regenerate the existing data points. Ie. the model may be overfitting the existing data.

In order to make sure a truly predictive model is made, existing datasets are typically split into training and testing datasets. The training dataset is used to make the model. The testing dataset is then used to test the predictive model to evaluate its predictive performance on completely new data.

The smaller the training set size, the less useful data is being used to generate the model. So, it may not be as accurate as it could be; i.e it may underfit the data. However, since the testing set is large, we can confidently state that the model is reliable if the testing data is accurately predicted.

On the other hand, if the training set is very large, there may not be sufficient testing data to confidently understand if the model is overfit or not.

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

Any machine learning algorithm has multiple hyper-parameters that can be used to 'tune' the model behaviour. For example, a Support Vector Machine method can be tuned with the kernel and C parameters. Some set of parameters may perform better than another at predicting a particular dataset. To efficiently and automaticaly test and extract the best combination of model parameters, the grid search technique may be used.

In the grid search technique, a list of the different sets of algorithm parameters is used as an input. The grid search technique then re-trains and compares cross-validation performance based on a chosen metric, for all combinations of algorithm parameters. The final result takes the form of a grid because each parameter forms a coordinate axis, and a cross-validation score is given for each combination of parameter 'coordinates'. The model with the highest score is then extracted from the grid search.

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer:

The typical method of training a learning model is to split the dataset into a training set and a smaller testing set. The testing set is set aside and solely used to ultimately evaluate the predictive ability of the model. The fraction of the training set is again set aside as the cross-validation set. This set is used to help tune model hyperparameters, eg. by using the grid search method. However there are two issues with the basic method:

1) Setting aside the testing set as well as the cross-validation set may result in too little data to adequately train a model. This is especially pertinent in problems where collecting data is expensive and hence only small datasets exist.

2) In the basic grid search method, the same training and cross-validation set is used to train multiple models/hyperparameter sets. This potentially introduces bias into the model, since the model is optimized to match the characteristics of one particular cross-validation set.

The K-fold cross-validation method helps mitigate the above two issues. The K-fold technique works by splitting the training dataset into k different randomized subsets/buckets. A series of k different combinations of the buckets then exist, with a different bucket acting as the cross-validation set in each combination. A model is then trained k different times with each training dataset combination. The final model/hyperparameter set score is based on an average of the k different model parameters.

The first benefit of this method is that the entire training dataset is used and thus, after the testing set is set aside, the entire remaining dataset is used in model generation. Second, since different cross-validation sets are used to initially evaluate the models, there is minimal chance of bias entering the model.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':list(range(1,11))}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,params,scoring=scoring_fnc,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Sensitivity¶

An optimal model is not necessarily a robust model. Sometimes, a model is either too complex or too simple to sufficiently generalize to new data. Sometimes, a model could use a learning algorithm that is not appropriate for the structure of the data given. Other times, the data itself could be too noisy or contain too few samples to allow a model to adequately capture the target variable i.e., the model is underfitted.
